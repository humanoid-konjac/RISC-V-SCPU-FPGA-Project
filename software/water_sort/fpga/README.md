# Water Sort FPGA 裸机固件

该目录提供 RV32I/ILP32 裸机启动、链接和双 COE 构建流程。默认构建 Step 5 完整游戏，复用上一级目录中经过主机单元测试的游戏核心。启动代码在进入游戏前执行 `isa_coverage.S`，确保最终应用实际包含并运行 CPU 已实现的全部 37 条指令。

完整固件轮询键盘事件 MMIO：左右键或 A/D 移动光标，Enter/Space 选择来源或倾倒，Esc 取消选择，R 重置。每次可见状态变化都会写入 8 根试管、UI 和步数 shadow 寄存器，然后通过 `COMMIT` 在帧边界整体显示。数码管显示十六进制步数；LED 显示当前光标，通关后全部点亮。

`phase3_main.c` 保留为之前的单色块链路诊断源文件，但不参与默认完整游戏构建。

## 构建

安装 GNU RISC-V 裸机工具链后执行：

```bash
make
```

默认生成：

- `coe/water_sort_game_i.coe`：加载到 `ROM_D`。
- `coe/water_sort_game_d.coe`：加载到 `RAM_B`。
- `build/water_sort_game.asm`：用于检查实际 RV32I 指令。

程序的 `.text` 位于 `0x0000_0000`，数据和栈位于 `0x1000_0000～0x1000_0fff`，并为向下增长的栈至少保留 512 字节。链接脚本会在指令超过 4 KiB 或数据侵占保留栈空间时失败。

2026-07-10 完整构建结果：`.text` 为 1396 字节、`.bss` 为 46 字节；反汇编包含全部 37 条已实现指令，真实 `SCPU` 固件仿真按固定关卡的已知解完成 21 步通关。
